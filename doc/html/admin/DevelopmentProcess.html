DITA Open Toolkit Development Process

The DITA Open Toolkit development process is modelled after other popular and successful Open Source projects, notably the Eclipse development process (for definitions and process statements).

Revision 2.0

Version 1.0 released February 27, 2005.

Version 2.0 released June 29, 2012.

Purpose and scope

This section describes the development process of DITA Open Toolkit (DITA-OT) project.

  • Project roles and responsibilities
    There are several roles in the DITA Open Toolkit open source project. Each has different rights and obligations.
  • DITA Open Toolkit Release management
    The DITA-OT project uses an agile development process, with roughly four-week development iterations followed by the release of a full test build that includes all new function developed up to that point. Roughly twice a year, a final "stable" build will be released.
  • Requesting features and reporting defects
    Defect reports and feature requests may be submitted at any time through the project page at github.
  • How to participate in the DITA Open Toolkit
    Any individual or any organization can contribute to the project; contributions must adhere to the existing toolkit license(s) and should fit in with the general goals of the project. All major contributors will get appropriate recognition in release announcements and on the project home page.
